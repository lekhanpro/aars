Retrieval That
Thinks Before It Retrieves

AARS uses LLM-based planner and reflection agents to dynamically select the optimal retrieval strategy for every query — keyword, vector, graph, or hybrid — then verifies sufficiency and retries if needed.

Why AARS?
Adaptive retrieval, not fixed pipelines
Traditional RAG uses one retrieval strategy for all queries. AARS introduces a metacognitive loop — analyzing each query before choosing how to retrieve.
🧠

Planner Agent

LLM classifies query type (factual, analytical, multi-hop) and selects the optimal strategy — keyword, vector, graph, or hybrid.

🔍

4 Retrieval Strategies

BM25 for exact terms, dense embeddings for semantic similarity, entity graph traversal for multi-hop, and hybrid fusion.

🔄

Reflection Loop

Evaluates if retrieved documents are sufficient. If not, it re-retrieves with a modified query and different strategy — up to 3 iterations.

⚖

RRF + MMR Fusion

Reciprocal Rank Fusion merges cross-strategy results. Maximal Marginal Relevance promotes diversity and reduces redundancy.

📄

Cited Answers

Grounded generation with inline citations, confidence scores, and full reasoning trace for every answer.

⚡

Production Ready

FastAPI + Docker Compose. Streamlit UI. Full test suite. Ready to deploy with one command.

Get Started
Deploy in 30 seconds
One command to run the full stack — API, vector store, and demo UI.
git clone https://github.com/lekhanpro/aars.git
cd aars
cp .env.example .env          # Add your ANTHROPIC_API_KEY
docker compose up --build -d   # Starts API :8000 + ChromaDB :8001 + UI :8501
