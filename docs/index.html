Adaptive retrieval that plans before it retrieves.

AARS is a query-aware RAG backend that selects retrieval strategy per question, checks whether the evidence is sufficient, and fuses keyword, vector, and graph results before answer generation.

No fake hosted demo. No fake benchmark claims. The runtime path and the benchmark in this repository are both real and rerunnable.

How It Works

Planning, reflection, and fusion are part of the runtime, not marketing copy.

  1. Plan Classify the query and choose keyword, vector, graph, hybrid, or none.
  2. Retrieve Run collection-aware retrieval across the configured stores.
  3. Reflect Check evidence sufficiency before answering from weak context.
  4. Retry Revise query and strategy when the first pass is not enough.
  5. Fuse Merge ranked lists with reciprocal-rank fusion and optionally rerank with MMR.
  6. Generate Return answer, citations, documents, confidence, and trace data.

Why AARS

Fixed-pipeline RAG is the wrong abstraction for mixed question types.

Query-aware strategy selection

Factual questions, semantic questions, and multi-hop questions do not get forced through the same retriever path.

Shared runtime state

Startup initializes shared orchestrator, ingestion, keyword, and graph state so the API does not rebuild core components per request.

Real fallback behavior

Sentence-transformers and spaCy are optional. AARS still runs with deterministic hashing embeddings and lightweight entity extraction.

Grounded outputs

Query responses can include citations, documents, plan, reflection results, and a bounded execution trace.

Benchmark

A reproducible local benchmark, not a hand-waved leaderboard claim.

The checked-in benchmark is the local offline fixture only: 12 documents, 9 questions, 7 systems, and no external dataset download. It exists for regression checking and for proving the retrieval runtime actually works.

Quick Start

Install it, benchmark it, run it.

Install

git clone https://github.com/lekhanpro/aars.git
cd aars
pip install -e ".[dev,ui]"

Run benchmark

python benchmarks/runner.py --output benchmarks/results_local.json

Start API

docker run -p 8001:8000 chromadb/chroma:latest
uvicorn src.main:app --host 0.0.0.0 --port 8000 --reload

Query example

curl -X POST http://localhost:8000/api/v1/query \
  -H "Content-Type: application/json" \
  -d '{
    "query": "What sparse ranking algorithm rewards exact term overlap?",
    "collection": "demo",
    "top_k": 5,
    "enable_planner": true,
    "enable_reflection": true,
    "enable_fusion": true,
    "enable_mmr": true,
    "enable_keyword": true,
    "enable_graph": true,
    "enable_trace": true
  }'
